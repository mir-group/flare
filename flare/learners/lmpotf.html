On-the-Fly Training in LAMMPS

class flare.learners.lmpotf.LMPOTF(sparse_gp: ~flare.bffs.sgp._C_flare.SparseGP, descriptors: ~typing.List, rcut: float, type2number: ~typing.Union[int, ~typing.List[int]], dftcalc: object, energy_correction: ~typing.List[float] = 0.0, force_training=True, energy_training=True, stress_training=True, dft_call_threshold: float = 0.005, dft_add_threshold: float = 0.0025, dft_xyz_fname: ~typing.Optional[str] = None, std_xyz_fname: ~typing.Optional[str] = None, model_fname: str = 'otf.flare', hyperparameter_optimization: ~typing.Callable[[~flare.learners.lmpotf.LMPOTF, object, int], bool] = <function LMPOTF.<lambda>>, opt_bounds: ~typing.Optional[~typing.List[float]] = None, opt_method: ~typing.Optional[str] = 'L-BFGS-B', opt_iterations: ~typing.Optional[int] = 50, post_dft_callback: ~typing.Callable[[~flare.learners.lmpotf.LMPOTF, int], None] = <function LMPOTF.<lambda>>, wandb: ~typing.Optional[object] = None, log_fname: str = 'otf.log')

Module for performing On-The-Fly (OTF) training, also known as active learning, entirely within LAMMPS.

Parameters

  • sparse_gp – The SparseGP object to train.

  • descriptors – A list of descriptor objects, or a single descriptor (most common), e.g. B2.

  • rcut – The interaction cut-off radius.

  • type2number – The atomic numbers of all LAMMPS types.

  • dftcalc – An ASE calculator, e.g. Espresso.

  • energy_correction – Per-type corrections to the DFT potential energy.

  • dft_call_threshold – Uncertainty threshold for whether to call DFT.

  • dft_add_threshold – Uncertainty threshold for whether to add an atom to the training set.

  • dft_xyz_fname – Name of the file in which to save the DFT results. Should contain ‘*’, which will be replaced with the current step.

  • std_xyz_fname – Name of the file in which to save ASE Atoms with per-atom uncertainties as charges. Should contain ‘*’, which will be replaced with the current step.

  • model_fname – Name of the saved model, must correspond to pair_coeff.

  • hyperparameter_optimization – Boolean function that determines whether to run hyperparameter optimization, as a function of this LMPOTF object, the LAMMPS instance and the current step.

  • opt_bounds – Bounds for the hyperparameter optimization.

  • opt_method – Algorithm for the hyperparameter optimization.

  • opt_iterations – Max number of iterations for the hyperparameter optimization.

  • post_dft_callback – A function that is called after every DFT call. Receives this LMPOTF object and the current step.

  • wandb – The wandb object, which should already be initialized.

  • log_fname – An output file to which logging info is written.

step(lmpptr, evflag=0)

Function called by LAMMPS at every step. This is the function that must be called by fix python/invoke.

Parameters

  • lmpptr (ptr) – Pointer to running LAMMPS instance.

  • evflag (int) – evflag given by LAMMPS, ignored.
